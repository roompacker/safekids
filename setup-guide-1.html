SafeKids Firebase 설정 가이드

1단계: Firebase 프로젝트 만들고 앱에 연결하기

📋 완료 체크리스트

  • Firebase 프로젝트 생성

  • GitHub Pages에 배포

1 Firebase 프로젝트 만들기

  1. Firebase Console 열기 →
  2. 구글 계정으로 로그인
  3. 프로젝트 추가 버튼 클릭
  4. 프로젝트 이름: safekids 입력
  5. Google 애널리틱스 → 사용 안함 (꺼도 됩니다)
  6. 프로젝트 만들기 클릭
  7. 잠시 기다리면 프로젝트 생성 완료!
💡 RoomPacker 때처럼 Firebase Console에서 진행해요. 이미 해보셨으니 익숙하실 거예요!

2 웹 앱 등록하기

  1. 프로젝트 메인 화면에서 </> (웹) 아이콘 클릭
  2. 앱 닉네임: SafeKids Web
  3. Firebase Hosting 체크는 ❌ 하지 마세요 (GitHub Pages 사용)
  4. 앱 등록 클릭
  5. 화면에 나오는 설정 코드를 복사합니다:
// 이런 형태의 코드가 나옵니다: const firebaseConfig = { apiKey: "AIzaSy...........", authDomain: "safekids-xxxxx.firebaseapp.com", projectId: "safekids-xxxxx", storageBucket: "safekids-xxxxx.appspot.com", messagingSenderId: "123456789", appId: "1:123456789:web:abcdef" };
⚠️ 이 코드를 반드시 복사해서 메모장에 저장해두세요! 나중에 index.html에 붙여넣을 거예요.

3 이메일 로그인 활성화

  1. 왼쪽 메뉴에서 빌드 → Authentication 클릭
  2. 시작하기 버튼 클릭
  3. Sign-in method 탭 클릭
  4. 이메일/비밀번호 클릭
  5. 첫 번째 토글 사용 설정 → ON으로 변경
  6. 저장 클릭
💡 이렇게 하면 부모님이 이메일+비밀번호로 가입/로그인할 수 있어요

4 Firestore Database 만들기

  1. 왼쪽 메뉴에서 빌드 → Firestore Database 클릭
  2. 데이터베이스 만들기 클릭
  3. 위치: asia-northeast3 (서울) 선택
  4. 테스트 모드에서 시작 선택 → 다음
  5. 만들기 클릭

5 Firestore 보안 규칙 설정

Firestore Database 화면에서 규칙 탭을 클릭하고, 아래 규칙으로 교체하세요:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // 사용자 프로필: 본인만 읽기/쓰기 match /users/{userId} { allow read, write: if request.auth != null && request.auth.uid == userId; } // 가족 그룹: 멤버만 접근 match /families/{familyId} { allow read, write: if request.auth != null && request.auth.uid in resource.data.members; allow create: if request.auth != null; // 가족 하위 컬렉션 (자녀, 장소, 일정, 알림, 위치) match /{subcollection}/{docId} { allow read, write: if request.auth != null; } } } }

게시 버튼을 눌러 저장하세요.

⚠️ 테스트 모드 규칙은 30일 후 만료됩니다. 위 규칙으로 꼭 교체하세요!

6 index.html에 Firebase 설정 넣기

index.html 파일을 열어서, 상단에 있는 이 부분을 찾으세요:

// 🔥 Firebase 설정 - 여기에 본인의 Firebase 설정을 넣으세요! const firebaseConfig = { apiKey: "YOUR_API_KEY", authDomain: "YOUR_PROJECT.firebaseapp.com", projectId: "YOUR_PROJECT_ID", storageBucket: "YOUR_PROJECT.appspot.com", messagingSenderId: "YOUR_SENDER_ID", appId: "YOUR_APP_ID" };

↓ 2단계에서 복사한 실제 Firebase 설정값으로 바꿔주세요:

// 예시 (본인 값으로 교체!) const firebaseConfig = { apiKey: "AIzaSyBxxxxxxxxxxxxxx", authDomain: "safekids-12345.firebaseapp.com", projectId: "safekids-12345", storageBucket: "safekids-12345.appspot.com", messagingSenderId: "987654321", appId: "1:987654321:web:abc123" };

7 GitHub Pages에 배포하기

RoomPacker처럼 GitHub에 올려서 배포합니다:

  1. GitHub에서 새 저장소(Repository) 만들기
    → 이름: safekids
  2. Settings → Pages → Source를 main 브랜치로 설정
  3. 아래 파일들을 업로드:
    • index.html (메인 앱)
    • manifest.json (PWA 설정)
    • setup-guide.html (이 가이드)
  4. 1~2분 후 https://[아이디].github.io/safekids/ 에서 접속 가능!
💡 RoomPacker 올릴 때랑 동일한 방법이에요!

📊 Firestore 데이터 구조

앱에서 자동으로 아래 구조로 데이터가 저장됩니다:

📁 users/{userId} ├─ name: "홍길동" ├─ email: "hong@email.com" ├─ role: "parent" └─ familyId: "가족ID" 📁 families/{familyId} ├─ name: "홍길동의 가족" ├─ members: [userId1, userId2] │ ├─ 📁 children/{childId} │ ├─ name: "지우" │ ├─ avatar: "👧" │ ├─ grade: "초3" │ ├─ status: "safe" │ └─ currentPlace: "영종초등학교" │ ├─ 📁 places/{placeId} │ ├─ name: "영종초등학교" │ ├─ type: "school" │ ├─ lat: 37.4932 │ ├─ lng: 126.4928 │ └─ radius: 150 │ ├─ 📁 schedules/{scheduleId} │ ├─ label: "등교" │ ├─ placeId: "장소ID" │ ├─ childId: "자녀ID" │ ├─ arriveBy: "08:30" │ ├─ leaveAfter: "14:30" │ └─ days: [1,2,3,4,5] │ ├─ 📁 alerts/{alertId} │ ├─ type: "arrive" │ ├─ childId: "자녀ID" │ ├─ title: "지우 - 학교 도착" │ ├─ message: "08:25에 도착" │ └─ createdAt: Timestamp │ └─ 📁 locations/{locationId} ← (2단계에서 추가) ├─ childId: "자녀ID" ├─ lat: 37.4950 ├─ lng: 126.4200 └─ timestamp: Timestamp
